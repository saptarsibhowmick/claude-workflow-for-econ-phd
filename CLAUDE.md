## CONSTITUTIONAL ARTICLES — IMMUTABLE

Article I: FILE SAFETY
1. NEVER delete any file. No exceptions.
2. NEVER modify any file in place. No exceptions.
3. NEVER overwrite any file. No exceptions.
4. To update a file: MOVE the existing version to `_deprecated/` in the same directory (create if needed) with prefix `YYYY-MM-DD_`, then CREATE the new version fresh at the original path, then APPEND an entry to `_deprecation-log.md` in the same directory.
5. Files in `data/raw/`, `data/resources/`, `code/replication/`, `correspondence/`, and `session-logs/` are PERMANENTLY IMMUTABLE. Never move, modify, or deprecate them.
6. For append-only files (`missing-lit.md`, `research-ideas.md`, `bibliography.bib`, `_deprecation-log.md`): read content, deprecate, create fresh with new content prepended to old. Log as "append operation."
7. Before re-running analysis code, deprecate all files in `output/tables/`, `output/figures/`, `output/logs/`.
8. If unsure whether a file should be modified, ASK THE AUTHOR.

Article II: DATA SENSITIVITY
1. NEVER read, open, or reference any file in `data/raw/restricted/`.
2. PII is NEVER included in session logs, MEMORY.md, correspondence, or any file outside `data/`.
3. Git commits NEVER include restricted data files.
4. If you encounter PII in any file, flag it and stop processing.

Article III: IDENTIFICATION FIRST
No estimation code is written until a strategy memo exists in `pre-analysis-plan/strategy-memo.md`.

Article IV: ADVERSARIAL INDEPENDENCE
Referee 2 and Blindspot audits run in a fresh terminal. The auditing instance never accesses the authoring instance's history.

Article V: CROSS-LANGUAGE VERIFICATION
Every headline result is replicated in at least one additional language before writing begins.

Article VI: DESIGN BEFORE RESULTS
During estimation, focus on specification correctness. Do not interpret point estimates until the design is intentional and documented.

Article VII: DOCUMENTATION TRAIL
Every audit produces a dated report. Every response is filed. Every session produces a log.

Article VIII: INTELLECTUAL PROPERTY
All research ideas, strategies, findings, and manuscripts are confidential. Never expose project content in search queries, commit messages, external services, or any public-facing context. Strategy memos are maximally sensitive — being scooped is a career-level risk. See `.claude/rules/privacy.md`.
